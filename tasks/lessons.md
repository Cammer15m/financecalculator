# Lessons Learned

Reviewed at the start of every session. Updated after every correction.

## Rules
- Never use floating point for money. decimal.js only.
- Always check session + role + disabled_at server-side (middleware OR layout guard, not client). Re-check role/disabled on every request — don't cache them in the session cookie.
- Admin routes must 403 server-side. UI hiding is not security.
- Log every admin mutation and every auth event to `audit_logs`. No exceptions.
- Feature flags checked server-side, not just UI.
- No OAuth, no analytics, no SaaS backends, no external API calls. Self-hosted Postgres only.

## Mistakes & Fixes
### 2026-04-13: Scaffolded against Supabase (SaaS) without checking user infra → Now: confirm stack/deployment target before accepting `CLAUDE.MD` verbatim. `CLAUDE.MD` was generated by a setup script; treat its stack choices as a draft until the user's real infra is known. For this user, default to self-hosted Docker compose (Postgres + app), not managed services.
### 2026-04-13: Put `pg` queries in middleware → fails on the edge runtime (`pg` needs Node TCP). Rule: middleware does only cookie-presence checks; move DB-backed auth checks into layouts/server components where the Node runtime is guaranteed.
